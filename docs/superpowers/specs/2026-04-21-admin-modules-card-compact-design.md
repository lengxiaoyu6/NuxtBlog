# 后台模块列表卡片紧凑化设计

## 背景

后台模块插件列表页当前卡片尺寸偏大，单屏可见信息较少。当前页面已经具备完整的模块名称、状态标签、时间信息与详情入口，交互层面无需调整，本次仅收缩视觉占位。

## 目标

缩小 `app/pages/admin/modules/index.vue` 中模块卡片的视觉高度与宽度占比，同时保持以下内容不变：

1. 模块名称、版本、描述与状态信息完整展示。
2. 查看详情、安装、启用、停用、卸载等行为保持原有逻辑。
3. 中小屏仍保持良好可读性，大屏单行容纳更多卡片。

## 方案

### 卡片容器

模块列表栅格由当前较宽松的双列布局调整为更紧凑的响应式布局：

1. 保留默认单列。
2. 在 `lg` 断点启用双列。
3. 在 `2xl` 断点启用三列。
4. 栅格容器增加 `justify-items-start`，避免模块数量较少时单卡撑满整列。

这样可以在大屏后台界面中降低单卡占比，减少空白区域。

### 卡片尺寸

卡片外层样式收缩为较小内边距和略小圆角，保留当前边框、阴影和悬停效果。

建议调整内容：

1. 外层内边距由 `p-6` 收缩为 `p-4 sm:p-5`。
2. 圆角由 `rounded-[2rem]` 收缩为 `rounded-[1.5rem]`。
3. 卡片主内容纵向间距由 `gap-5` 收缩为 `gap-4`。
4. 卡片外层增加 `w-full max-w-[26rem]`，将单卡最大宽度限制在约 `416px`。

### 卡片头部

头部图标与文本区同步紧凑化：

1. 图标容器由 `h-12 w-12 rounded-2xl` 收缩为 `h-10 w-10 rounded-xl`。
2. 图标尺寸由 `22` 收缩为 `18`。
3. 标题与副标题的分组间距由 `gap-4` 收缩为 `gap-3`。

模块名称字号、状态标签字号与按钮高度保持现状，避免信息辨识度下降。

### 信息区

时间信息区保持现有双列信息结构，仅压缩容器内边距与整体间距：

1. 信息区容器内边距由 `p-4` 收缩为 `p-3`。
2. 信息区网格间距由 `gap-4` 收缩为 `gap-3`。

### 测试策略

先修改 `tests/app/admin-module-center-ui.test.ts`，增加对紧凑卡片关键类名的断言，使测试先失败；再修改 `app/pages/admin/modules/index.vue` 使测试通过。

断言范围包括：

1. 列表栅格包含 `lg:grid-cols-2` 与 `2xl:grid-cols-3`。
2. 列表栅格包含 `justify-items-start`。
3. 卡片容器包含 `w-full max-w-[26rem]`、`rounded-[1.5rem]` 与 `p-4 sm:p-5`。
4. 图标容器包含 `h-10 w-10 rounded-xl`。

## 影响范围

仅涉及模块插件列表页的视觉样式与对应静态测试，不影响接口、数据结构、模块状态变更逻辑与详情页。
